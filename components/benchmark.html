Benchmark Overview

ParallelBench consists of 17 tasks across 3 categories. Each task exposes different aspects of the speed-quality trade-off in dLLMs.

Waiting Line

Waiting Line is a set of list operations (e.g., shuffle, replace) manipulating realistic customer service scenarios. The number of customers n serves as a controlled variable for adjusting difficulty.

Text Writing

Text Writing consists of five text generation tasks: summarization, paraphrasing, words-to-sentence generation. We introduce a Grammar Score metric that imposes stronger token-level dependencies.

5 tasks

Puzzles

Puzzles consists of two classic puzzle tasks: Sudoku, which has a unique solution, and Latin Square, which has multiple valid solutions. This difference in solution space affects parallel decoding differently based on conditional total correlation C(Y|X).
