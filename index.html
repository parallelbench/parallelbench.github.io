StructEval: Benchmarking LLMs' Capabilities to Generate Structural Outputs

StructEval is a comprehensive benchmark that assesses large language models on producing structured outputs across text-only and visually rendered formats. We evaluate 18 formats and 44 task types spanning generation and conversion scenarios.

Abstract

As Large Language Models (LLMs) become integral to software development workflows, their ability to generate structured outputs has become critically important. We introduce StructEval, a comprehensive benchmark for evaluating LLMs' capabilities in producing both non-renderable (JSON, YAML, CSV) and renderable (HTML, React, SVG) structured formats. Unlike prior benchmarks, StructEval systematically evaluates structural fidelity across diverse formats through two paradigms: (1) generation tasks, producing structured output from natural language prompts, and (2) conversion tasks, translating between structured formats.

Our benchmark encompasses 18 formats and 44 types of tasks, with novel metrics for format adherence and structural correctness. Results reveal significant performance gaps‚Äîeven state-of-the-art models like o1-mini achieve only 75.58% average score, with open-source alternatives lagging approximately 10 points behind. We find generation tasks more challenging than conversion tasks, and producing correct visual content more difficult than generating text-only structures.

Evaluation Framework

Our evaluation framework employs four core metrics:

üîß Render Score (T/V)

Binary metric (0 or 1) indicating whether the generated code can be successfully loaded or rendered without syntax errors.

‚úì Syntax Score (T)

Verifies structural correctness using dot-path rules and calculates the percentage of satisfied constraints.

üîç Keyword Matching (V)

Evaluates the presence of desired keywords using exact string matching on raw generated output code.

üëÅÔ∏è VQA Score (V)

Assesses visual correctness of rendered content through question-answer pairs about the generated output.

Score Aggregation Methods

üé® Renderable Formats

HTML, React, SVG, LaTeX, Mermaid, etc.

Final Score = (0.2 √ó Render Score) +
             (0.1 √ó Keyword Matching) +
             (0.7 √ó VQA Score)

üìù Non-Renderable Formats

JSON, XML, YAML, CSV, TOML

Final Score = (0.2 √ó Render Score) +
             (0.8 √ó Syntax Score)

Performance Analysis

Performance by Task Type

StructEval-V generation tasks remain the most challenging category, with even the strongest proprietary models achieving less than 70% accuracy. Conversion tasks in StructEval-T exhibit comparatively higher stability, yet still reveal gaps in syntactic adherence among mid-sized open models.

Challenging Formats (Average < 50%)

TikZ, Typst, and Canvas-based graphics frequently fall below the 50% threshold due to brittle rendering pipelines and precise spatial requirements. Benchmark difficulty highlights the need for robust visual reasoning and code generation competencies.

üìâ Performance Gap

Even state-of-the-art models struggle with structured output generation. GPT-4o attains only 76.02% average score, while the top open-source model (Qwen3-4B) trails at 67.04%.

‚öñÔ∏è Task Difficulty

Generation tasks are broadly more challenging than conversion tasks. Visual rendering (StructEval-V) proves consistently harder than text-only structures (StructEval-T).

‚ö†Ô∏è Challenging Formats

Formats involving rich layout specification‚Äîsuch as Mermaid diagrams, Matplotlib charts, and Vega visualizations‚Äîdrive significant variance across models, underscoring fragile tool-use capabilities.

üß™ Evaluation Implications

The benchmark motivates new research on tooling-augmented reasoning, ensemble VQA, and robust syntax enforcement to close the gap between closed and open ecosystems.
